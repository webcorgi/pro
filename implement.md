# 🧭 Task Execution Guidelines

## 📘 Overview

이 문서는 프로젝트의 모든 작업(`tasks.md`)을 수행할 때 따라야 하는 **필수 실행 지침**을 정의합니다.  
정확한 구현을 위해 반드시 `requirements.md`, `design.md`, `tasks.md` 세 파일을 함께 확인해야 합니다.

---

## ⚙️ 기본 실행 순서 (Execution Flow)

1. **요구사항 및 설계 파일 확인**
   - 작업을 시작하기 전에 반드시 `requirements.md`, `design.md`, `tasks.md` 파일을 **모두 읽어야 합니다.**
   - 이 문서들을 확인하지 않으면 **부정확한 구현**이 발생할 수 있습니다.

2. **작업(Task) 선택**
   - `tasks.md` 파일의 **작업 목록(Task List)** 을 확인합니다.
   - 요청된 작업에 **하위 작업(Sub-task)** 이 있다면, 반드시 **하위 작업부터 순서대로** 진행합니다.

3. **단일 작업 원칙**
   - **한 번에 하나의 작업만 수행합니다.**
   - 다른 작업의 기능을 미리 구현하지 않습니다.

4. **요구사항 검증**
   - 작업 완료 후, 요구사항 또는 세부 사양에 따라 결과를 검증합니다.

5. **작업 완료 후 보고**
   - 작업이 완료되면 `tasks.md` 파일을 업데이트합니다.
   - **그 즉시 중단하고**, 다음 작업으로 자동 이동하지 않습니다.
   - 사용자의 검토 및 승인을 기다립니다.

---

## ⚠️ 중요 지침 (Critical Rules)

- ❌ **가정 금지:** 사용자의 선호나 요구사항을 추측하지 말고, 반드시 명시적으로 물어야 합니다.  
- 🪜 **단계 추적:** 현재 수행 중인 단계를 명확히 기록해야 합니다.  
- 🔒 **단계 병합 금지:** 여러 단계를 한 번에 처리하지 말고, 반드시 **한 단계씩 분리**해서 실행합니다.  
- 🧩 **단일 작업 원칙:** 오직 하나의 작업만 수행하고, 완료되면 **다음 작업으로 자동 진행하지 않습니다.**

---

## ✅ 올바른 실행 예시

1. requirements.md, design.md, tasks.md 파일 모두 확인
2. tasks.md에서 "Task #3: User Login API" 확인
3. 하위 작업 존재 시 -> Sub-task #3.1부터 수행
4. 구현 완료 후 테스트 및 검증
5. 결과를 tasks.md에 기록하고 중단
6. 사용자에게 “Task #3 완료. 다음 단계로 진행할까요?”라고 문의

---

## 🚫 잘못된 실행 예시
- requirements.md를 확인하지 않고 바로 코딩 시작
- Task #3과 Task #4를 동시에 구현
- 사용자의 승인 없이 다음 단계로 자동 진행
- 수정 피드백 반영 없이 문서 업데이트


---

**핵심 요약**

> 🔹 한 번에 하나의 작업만 수행
> 🔹 모든 요구사항 및 설계 문서를 반드시 검토
> 🔹 작업 완료 후 사용자 승인 없이 절대 다음 단계로 이동 금지
